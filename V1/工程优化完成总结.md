## ✅ 已完成的优化任务

### 1. 🧹 清理废弃代码

**文件**: `pages/tool_dunjia.py`

**清理内容**: 删除第674行之后约270行的废弃代码，包括：

- 旧版参考信息区 `_create_reference_area()`

**效果**: 文件从 943 行精简到 673 行，减少 **28.6%** 的冗余代码。

---

### 2. 🏗️ 提取页面基类

**新增文件**: `pages/base_tool.py`

**BaseToolPage 基类提供的公共方法**:

| 方法 | 功能 | 原重复次数 |
|------|------|-----------|
| `_show_result()` | 在 CTkTextbox 中显示结果 | 8/10 个文件 |
| `_show_error()` | 在 CTkTextbox 中显示错误信息 | 8/10 个文件 |
| `_bind_mousewheel()` | 阻止嵌套滚动区滚轮事件冒泡 | 5/10 个文件 |
| `_parse_int()` | 安全解析整数 | 4/10 个文件 |
| `_parse_float_or_inf()` | 解析浮点数或无穷大 | 3/10 个文件 |
| `_fmt()` | 格式化数字显示 | 4/10 个文件 |

**基类特性**:
- 统一的 `__init__(self, parent, colors=None)` 签名
- 可覆盖的 `_error_prefix` 属性（默认 "⚠"）
- 抽象方法 `_build_ui()` 强制子类实现
- 完善的文档字符串和类型注解

**效果**: 所有工具页面现在可以继承 `BaseToolPage`，复用这些公共方法，减少约 **60-80 行** 重复代码/文件。

    def _calculate(self):
        val = self._parse_int(self.entry.get(), 0)
        if val < 0:
            self._show_error(self.result_box, "输入无效")
        else:
            self._show_result(self.result_box, f"结果: {val}")
