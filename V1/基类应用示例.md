# 基类应用示例 - 如何改造现有工具页面

## 改造前后对比

以 `tool_b.py` (淬炼计算器) 为例，展示如何应用基类优化代码。

---

## 改造前（原始代码）

```python
import customtkinter as ctk

class ToolBPage(ctk.CTkFrame):
    """装备淬炼计算器界面"""
    
    def __init__(self, parent, colors: dict):
        super().__init__(parent, fg_color="transparent")
        self.colors = colors
        self._build_ui()
    
    def _show_result(self, textbox: ctk.CTkTextbox, text: str):
        """在结果区域显示文本"""
        textbox.configure(state="normal")
        textbox.delete("1.0", "end")
        textbox.insert("1.0", text)
        textbox.configure(state="disabled")

    def _show_error(self, textbox: ctk.CTkTextbox, msg: str):
        """在结果区域显示错误信息"""
        self._show_result(textbox, f"⚠️ {msg}\n")
    
    # ... 其他方法
```

**问题**:
- `_show_result()` 和 `_show_error()` 在 8 个工具页面中重复
- 每个文件都要维护这些相同的方法
- 代码冗余约 **20-30 行/文件**

## 改造后（使用基类）

```python
import customtkinter as ctk
from pages.base_tool import BaseToolPage  # 导入基类

class ToolBPage(BaseToolPage):  # 继承 BaseToolPage 而不是 ctk.CTkFrame
    """装备淬炼计算器界面"""
    
    # 可选：自定义错误前缀
    _error_prefix = "⚠️"
    
    def __init__(self, parent, colors: dict = None):
        super().__init__(parent, colors)  # 基类处理初始化
        # 如果有特殊的数据加载，在这里进行
        self._build_ui()
    
    # 删除 _show_result() 方法（使用基类提供的）
    # 删除 _show_error() 方法（使用基类提供的）
    
    # ... 其他方法保持不变
```

**改进**:
- 删除 **20-30 行** 重复代码
- 统一错误处理风格
- 如需修改行为，只需覆盖基类方法

## 完整改造示例

### 改造前的计算方法

```python
def _calculate_max_level(self):
    """根据材料量计算可达成淬炼等级"""
    try:
        # 读取输入
        mat1 = int(self.mat1_entry.get() or "0")
        mat2 = int(self.mat2_entry.get() or "0")
        
        if mat1 < 0 or mat2 < 0:
            # 手动实现错误显示
            textbox = self.result1_box
            textbox.configure(state="normal")
            textbox.delete("1.0", "end")
            textbox.insert("1.0", "⚠️ 材料数量不能为负数\n")
            textbox.configure(state="disabled")
            return
        
        # 计算逻辑
        result = self._do_calc(mat1, mat2)
        
        # 手动实现结果显示
        textbox = self.result1_box
        textbox.configure(state="normal")
        textbox.delete("1.0", "end")
        textbox.insert("1.0", f"✅ 计算完成\n\n结果: {result}")
        textbox.configure(state="disabled")
        
    except ValueError:
        textbox = self.result1_box
        textbox.configure(state="normal")
        textbox.delete("1.0", "end")
        textbox.insert("1.0", "⚠️ 请输入有效数字\n")
        textbox.configure(state="disabled")
```

**问题**:
- 每次显示结果/错误都需要 5 行代码
- 代码重复，难以维护
- 容易遗漏 `configure(state=...)`

---

### 改造后的计算方法

```python
def _calculate_max_level(self):
    """根据材料量计算可达成淬炼等级"""
    try:
        # 使用基类的 _parse_int 安全解析
        mat1 = self._parse_int(self.mat1_entry.get(), 0)
        mat2 = self._parse_int(self.mat2_entry.get(), 0)
        
        if mat1 < 0 or mat2 < 0:
            # 使用基类的 _show_error 显示错误
            self._show_error(self.result1_box, "材料数量不能为负数")
            return
        
        # 计算逻辑
        result = self._do_calc(mat1, mat2)
        
        # 使用基类的 _show_result 显示结果
        output = f"✅ 计算完成\n\n结果: {result}"
        self._show_result(self.result1_box, output)
        
    except Exception as e:
        # 使用基类的 _show_error 显示异常
        self._show_error(self.result1_box, f"计算出错: {str(e)}")
```

**改进**:
- 代码行数从 **30 行减少到 15 行**
- 逻辑更清晰，可读性更强
- 统一的错误处理风格

## 滚轮事件冒泡处理示例

### 改造前

```python
def _build_ui(self):
    # ... 创建 CTkTextbox
    self.result_box = ctk.CTkTextbox(...)
    
    # 手动绑定滚轮事件
    def on_mousewheel(event):
        self.result_box._parent_canvas.yview_scroll(int(-1 * (event.delta / 120)), "units")
        return "break"
    
    self.result_box.bind("<MouseWheel>", on_mousewheel, add="+")
    self.result_box.bind("<Button-4>", lambda e: (self.result_box._parent_canvas.yview_scroll(-1, "units"), "break")[1], add="+")
    self.result_box.bind("<Button-5>", lambda e: (self.result_box._parent_canvas.yview_scroll(1, "units"), "break")[1], add="+")
```

**问题**: 每个 textbox 都要重复 10 行代码

---

### 改造后

```python
def _build_ui(self):
    # ... 创建 CTkTextbox
    self.result_box = ctk.CTkTextbox(...)
    
    # 使用基类方法，一行搞定
    self._bind_mousewheel(self.result_box)
```

**改进**: **10 行代码 → 1 行**

## 常见问题

### Q1: 基类方法不满足需求怎么办？

**A**: 在子类中覆盖基类方法即可。

```python
class ToolXxxPage(BaseToolPage):
    # 覆盖基类的 _show_error，添加音效
    def _show_error(self, textbox, msg):
        super()._show_error(textbox, msg)
        self.play_error_sound()  # 额外逻辑
```

### Q2: 是否必须使用所有基类方法？

**A**: 不必须。继承基类后，可以选择性使用其提供的方法。

### Q3: 如何测试基类改造是否成功？

**A**: 
1. 运行程序，进入改造后的工具页面
2. 触发正常计算流程
3. 触发错误场景（输入非法值）
4. 测试滚轮滚动是否正常
