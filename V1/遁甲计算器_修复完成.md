## 💡 改进的计算逻辑

### 模式1：根据材料计算可达等级
```python
def _calc_reachable_level(start_grade, start_level, materials):
    """
    1. 从当前品阶和等级开始
    2. 逐级检查材料是否充足
    3. 材料充足则扣除并升级
    4. 材料不足则停止
    5. 品阶满级自动进阶到下一品阶
    6. 返回最终状态 + 逐级路径 + 材料使用汇总
    """
```

**输出示例**：
- 起始状态：黄阶 Lv.0
- 可达状态：天阶 Lv.80
- 材料使用：
  - 古迹灵石：36 个
  - 古迹灵石·玄：49 个
  - 古迹灵石·地：64 个
  - 古迹灵石·天：80 个

### 模式2：根据目标计算所需材料
```python
def _calc_required_materials(start_grade, start_level, end_grade, end_level):
    """
    1. 确定起始和结束的品阶索引
    2. 逐品阶、逐级累加材料消耗

    4. 分品阶统计材料需求
    5. 返回逐级路径 + 材料需求汇总
    """
```

**输出示例**：

## 🚀 使用方法

### 场景1：我有100个古迹灵石·天，从天阶0级能练到几级？

1. 选择 **"📊 根据材料计算等级"**
2. 设置：当前品阶 = 天阶，当前等级 = 0
3. 输入：古迹灵石·天 = 100
4. 点击 **"▶ 计算可达到的遁甲等级"**

**结果**：
- 可达状态：天阶 Lv.80
- 使用材料：古迹灵石·天 100个
- 详细路径表格显示
