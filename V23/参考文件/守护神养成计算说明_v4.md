### 1.2 请神符产出规则

- 每次使用请神符：若抽到**已有**守护神 → 获得该守护神残念 **10个**
- 若抽到**未有**守护神 → 直接获得该守护神（不产出残念）
- 计算时默认已拥有所有守护神（即每次请神均产出残念10个）

### 1.3 残念精华转化规则

- 守护神达到**5星后**，后续获得的该守护神残念可分解为精华
- 精华**只能在同一神仙谱组内流转**（不跨组）
- 精华换算：`残念数 × 该守护神精华单价 = 获得精华数`
- 购买残念：`精华数 ÷ 目标守护神精华单价 = 可购买残念数`（取整）

- 每次请神 +100 幸运值
- 幸运值达到上限时，**必出组内高品质守护神（随机一个）**，然后幸运值清零
- 各组幸运值上限与保底守护神：

## 三、每张请神符期望残念计算（核心公式）

### 3.1 公式定义

```
每张符期望残念[守护神X] = 普通出货期望 + 幸运值保底贡献 + 次数保底贡献
```

**普通出货期望：**
```
普通出货期望 = (该守护神权重 / 总权重) × 10
```

**幸运值保底贡献（仅保底守护神享有）：**
```
幸运值保底贡献 = (1 / 保底守护神数量) × 10 / 保底触发次数
```
- 保底触发次数 = 幸运值上限 / 100
- 若该守护神不在保底列表，贡献 = 0

**次数保底贡献（仅四值功曹-值年功曹）：**
```
次数保底贡献 = 10 / 115
```

### 3.3 期望所需请神符公式

```python
import math

def fu_needed(nian_gap: int, exp_per_fu: float) -> int:
    """
    nian_gap   : 残念缺口（需要获取的残念数）
    exp_per_fu : 每张符期望残念（见3.2速查表）
    返回       : 期望所需请神符数（向上取整）
    """
    if nian_gap <= 0:
        return 0
    return math.ceil(nian_gap / exp_per_fu)
```

## 五、平均星级的分布展开算法

当输入为平均星级时，需展开为具体分布，输出**最乐观分布**和**最悲观分布**两种极端情况。

### 5.1 定义

**最乐观分布（消耗最少的情况）：**
- 含义：当前已养得尽可能好，缺口最小
- 分配原则：**从高到低填满**，高星多低星少
- 例：4人组平均3.5星（总星级=14）→ [5, 5, 2, 2]

**最悲观分布（消耗最多的情况）：**
- 含义：当前养得尽可能差，缺口最大
- 分配原则：**从低到高填满**，低星多高星少
- 例：4人组平均3.5星（总星级=14）→ 最极端如 [0, 4, 5, 5]

### 5.2 展开算法

```python
def expand_distribution(avg_star: float, n: int, mode: str) -> list[int]:
    total = round(avg_star * n)
    stars = [0] * n

    if mode == 'optimistic':
        for i in range(n):
            stars[i] = min(5, total)
            total -= stars[i]
        stars.sort(reverse=True)

    elif mode == 'pessimistic':
        remaining = total
        for i in range(n - 1, -1, -1):
            stars[i] = min(5, remaining)
            remaining -= stars[i]
        stars.sort()

    return stars
```

### 5.3 约束条件

- 每个守护神星级范围：**[0, 5]**
- 目标分布中每个守护神的目标星级 **≥ 当前对应守护神的星级**（不允许倒退）
- 若目标平均星级 ≤ 当前平均星级，直接返回消耗=0

### 5.4 当前+目标的组合规则

| 场景 | 当前分布 | 目标分布 | 说明 |
|------|---------|---------|------|
| **消耗最少** | 最乐观（当前已养很好）| 最悲观（目标只需均匀提升）| 缺口最小 |
| **消耗最多** | 最悲观（当前养得很差）| 最乐观（目标要求高星集中）| 缺口最大 |

## 六、核心计算逻辑

### 6.1 计算单个守护神的残念缺口

```python
def calc_nian_gap(cur_star: int, tgt_star: int, held_nian: int = 0) -> int:
    need = STAR_CUM[tgt_star] - STAR_CUM[cur_star]
    return max(0, need - held_nian)
```

### 6.2 残念缺口转换为请神符数量（修正版）

```python
# ⚠️ v4修正：不再使用 gap/10，改为基于真实期望残念
def gap_to_fu(nian_gap: int, god_name: str, group_name: str) -> int:
    exp_per_fu = EXP_MAP[group_name][god_name]  # 见3.2速查表
    return math.ceil(nian_gap / exp_per_fu)
```

### 6.3 组内精华流转计算

**触发条件：**
- 同组内存在已满5星的守护神（或本次计算后会满5星）
- 满5星后，持有的多余残念可分解为精华

**计算步骤：**

```
Step 1：按精华单价从低到高排序同组守护神

Step 2：逐个计算各守护神残念缺口（扣除直接持有残念）

Step 3：对目标为5星的守护神，计算多余残念→精华贡献：
        多余残念 = max(0, 直接持有残念 - 升星所需残念)
        贡献精华 = 多余残念 × 该守护神精华单价

Step 4：精华池 = 持有精华 + 所有贡献精华之和

Step 5：精华池优先补贴【精华单价最高】的守护神：
        可购买残念 = 精华池 ÷ 目标守护神精华单价（取整）
        减少对应守护神的残念缺口

Step 6：用修正后的 gap_to_fu() 计算各守护神最终请神符需求
```

**关键约束：**
1. 精华只在**同组内**流转，不跨组
2. 只有**满5星后**持有的多余残念才能分解产出精华
3. 精华流向优先级：精华单价最高的守护神优先

### 6.4 完整计算流程

```
输入：
  - 神仙谱组（确定组内守护神列表）
  - 当前状态：平均星级（简单）或各守护神星级（精确）
  - 目标状态：平均星级（简单）或各守护神星级（精确）
  - 可选：已持有残念、精华数量

处理：
  If 简单模式（平均星级输入）：
    展开为最乐观分布和最悲观分布（当前+目标各两种）
    组合为消耗最少方案和消耗最多方案

  If 精确模式（各守护神星级输入）：
    直接使用精确分布，只输出一个精确结果

  对每种方案：
    ① 计算各守护神残念缺口
    ② 应用组内精华流转（Step 3-5）
    ③ 用 gap_to_fu()（基于真实概率+保底）输出各守护神所需请神符

输出：
  [简单模式]
    - 消耗最少方案：当前/目标分布展开 + 各守护神请神符 + 精华流转 + 合计
    - 消耗最多方案：（同上格式）
    - 快速参考：消耗区间 [最少N张, 最多M张]

  [精确模式]
    - 各守护神所需请神符明细 + 精华流转说明 + 合计
```

## 八、边界情况处理

| 情况 | 处理方式 |
|------|---------|
| 目标星级 = 当前星级 | 输出消耗=0 |
| 目标平均星级 < 当前平均星级 | 报错提示，不计算 |
| 平均星级展开后无法满足"目标≥当前"约束 | 强制调整：目标 = max(目标, 当前) |
| 守护神已满5星仍输入为目标5星 | 消耗=0 |
| 精华池不足以购买完整1个残念 | 精华余量保留，不足部分仍需请神符 |
