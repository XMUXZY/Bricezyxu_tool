# 在线更新功能使用指南

## 概述

本工具已内置**在线自动更新**机制，当你将工具分享给其他人后，只需在服务器上发布新版本，用户即可一键更新到最新版本。

## 架构流程

```
┌──────────┐     请求 version.json      ┌───────────────┐
│  用户电脑  │ ─────────────────────────→ │ 远程服务器     │
│          │ ←───────────────────────── │               │
│          │   返回版本信息+下载地址       │ (GitHub/Gitee)│
│          │                             │               │
│  下载zip │ ─────────────────────────→ │               │
│  解压覆盖 │                             │               │
│  重启应用 │                             │               │
└──────────┘                             └───────────────┘
```

## 发布者操作（你需要做的）

### 第一步：准备远程服务器

选择以下任一方式托管文件：

#### 方案 A：GitHub / Gitee（推荐，免费）

1. **新建一个公开仓库**（如 `game-ops-toolkit-releases`）
2. **开启 GitHub Pages** 或直接用 raw 文件访问

3. 将以下文件推送到仓库：
   - `version.json` — 版本信息文件
   - `update.zip` — 更新包

4. 获取 raw 文件的 URL，例如：
   ```
   https://raw.githubusercontent.com/你的用户名/game-ops-toolkit-releases/main/version.json
   ```

#### 方案 B：自己的服务器/对象存储

将文件上传到任意可公开访问的 HTTP 服务器目录即可。

### 第二步：修改 updater.py 中的服务器地址

编辑 `V1/updater.py`，修改顶部的配置：

```python
UPDATE_SERVER_URL = "https://your-server.com/game-ops-toolkit/"
# 例如：
# UPDATE_SERVER_URL = "https://raw.githubusercontent.com/xxx/releases/main/"
# UPDATE_SERVER_URL = "https://oss.example.com/toolkit/"
```

### 第三步：打包更新包

每次发布新版本时：

1. **修改版本号** — 编辑 `updater.py` 中的 `CURRENT_VERSION`
2. **制作更新包 zip**：
   ```
   # 只需包含 V1 目录下的核心代码文件
   # 排除 __pycache__、临时文件等
   cd V1/
   # 使用脚本或手动打包，包含 main.py、pages/、updater.py 等
   ```
3. **更新 version.json**：

```json
{
    "version": "1.1.0",
    "release_date": "2026-04-14",
    "download_url": "https://你的服务器/update.zip",
    "changelog": [
        "新增：注灵养成计算器",
        "修复：已知bug"
    ],
    "force_update": false,
    "min_version": "1.0.0",
    "file_hash": "",
    "file_size": 0
}
```

4. **将 `version.json` 和 `update.zip` 上传到服务器**

### 第四步：分发初始包

首次分享时，将整个 `V1/` 文件夹（含 `updater.py`）打包发给用户。用户运行后即可自动检测后续更新。

---

## version.json 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `version` | string | 是 | 新版本号，如 `"1.1.0"` |
| `release_date` | string | 否 | 发布日期 |
| `download_url` | string | 否 | 下载地址（不填则拼接默认路径） |
| `changelog` | array | 否 | 更新日志列表 |
| `force_update` | bool | 否 | 是否强制更新（跳过按钮隐藏） |
| `min_version` | string | 否 | 最低兼容版本 |
| `file_hash` | string | 否 | SHA256 校验值（安全增强） |
| `file_size` | int | 否 | 文件大小字节（完整性校验） |

## 用户端体验

### 启动自动检查
- 应用启动后会在后台静默检查版本
- 如发现新版本，右下角弹出浮动提示（15秒自动消失）
- 点击"去更新"跳转到设置页

### 手动检查
- 进入 **设置** → **在线更新** → 点击 **"检查更新"** 按钮
- 显示当前版本和最新版本对比
- 有新版时弹出详情对话框

### 更新流程
1. 弹窗显示版本信息和更新日志
2. 点击 **"⬇️ 立即更新"**
3. 进度条显示下载状态
4. 下载完成后自动解压覆盖
5. 应用自动重启，运行新版本

---

## 安全性设计

1. **SHA256 校验**：可在 version.json 中配置 `file_hash`，下载完成后自动校验
2. **文件大小校验**：通过 `file_size` 防止下载不完整
3. **自动备份**：更新前会备份旧版到 `{app}_backup_时间戳/` 目录
4. **增量更新友好**：只覆盖变更的 Python 文件

## 注意事项

- `update.zip` 内应保持与本地相同的目录结构（根目录下有 `main.py`）
- 建议排除 `__pycache__`、`*.pyc`、`_update_cache` 等非必要文件
- 如果用户网络受限，可改用内网服务器地址
- 首次分发时确保 `updater.py` 随主程序一起分发
